Interface Framework for Roblox

HyperMenu is a high-performance, single-module GUI solution built specifically for Luau. Optimized for latency, controller support, and seamless integration into any experience.

The Origin Story

HyperMenu didn't start in a corporate office. It started in a Roblox Studio session during a 48-hour Game Jam. We were building a fast-paced combat game and realized that traditional GUIs just couldn't keep up with the frame-perfect inputs we needed.

The goal was simple: "One module, zero lag." We spent months rewriting the tweening engines and input listeners to bypass the usual Roblox overhead. Today, HyperMenu is the tool we wish we had when we first started developing.

Studio Setup

1

Import the Module

Place the HyperMenu folder inside ReplicatedStorage so it can be accessed by both Client and Server.

2

Initialize on Client

Inside a LocalScript in StarterPlayerScripts, initialize the framework.

-- Client Bootstrap local ReplicatedStorage = game:GetService("ReplicatedStorage") local HyperMenu = require(ReplicatedStorage:WaitForChild("HyperMenu")) HyperMenu.Init({ Theme = "Dark", Keybind = Enum.KeyCode.F4 })

Visual Configuration

Customize your theme by modifying the Settings.lua ModuleScript. All colors use standard Color3.fromRGB values.

return { AccentColor = Color3.fromRGB(0, 162, 255), BackgroundTransparency = 0.1, BlurStrength = 20, Font = Enum.Font.GothamBold, RoundedCorners = true }
